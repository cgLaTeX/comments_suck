Programming is entirely about managing complexity

Rule 0

Have a reason for everything you do.

  • Variable names
  • Class names
  • Class naming style (CamelCase)
  • Line spacing, commit messages, plot colors, etc.

Rule 1

Remove redundancy.

UPS is famous for having optimized everything; eliminating left turns, closing the truck door automatically, putting a pencil in your off-hand pocket, etc. It's extreme. Be this extreme in your writing! Don't just DRY (don't repeat yourself) your code, DRY your commit messages and comments, too! A commit message that says "changed file XYZ.cpp" is obvious—git tells you that already. Instead say "added logging to XYZ.cpp".
Change interpolate.cpp
Add logging to interpolate.cpp

Comments are evil.

									# Report simulation progress in 10% increments on stdout
									if(step % (time_idx_ubound / 10) == 0) {

									}

									# Abort simulation if NaN/+-inf detected
									if(!all_finite(step)) {
											const std::string msg = "unbounded history values at or before step ";
											throw std::domain_error(msg + std::to_string(step));
									}
								}
